## 一、物件 & VFX 切图（07-03，又踩了「等分切」的坑）

### 1.1 物件切图 + Assets 同步工作流

- 母版在 `art/objects/<category>/`（bridges / houses{128,256} / trees / farmlands / stones / fences / battlestages），同步进 `Assets/Art/Objects/<Capitalized>/`。每张 PNG 要手写 `.meta`（`textureType 8` sprite、PPU 64、Point filter、`alphaIsTransparency`、`alignment 7`=BottomCenter），`PixelArtImportPostprocessor` 重导入时再补 FullRect mesh。
- **透明底大图**：scipy `label` 连通域 + `binary_dilation(5-6)` 把碎石/多块簇**粘在一起**当一件，按 `y//band` 分行再按 x 排。
- **镂空物件（栅栏）连通域不靠谱**：栅栏是「栏杆 + 立柱 + 大缝」，膨胀会把相邻件粘死、又把一根栅栏拆成一堆条 → 只能**看降采样编号预览图人工挑**哪些连通域是栅栏（那张大图其实是整套村庄件：房子/树/石头/井/箱子/栅栏混在一起）。还要抠掉粘在栅栏上的绿叶（键掉「绿色占优」像素，栅栏是棕木不受影响）。
- **灰底大图（旧石头 / 战斗背景）不是真透明**：按背景色做**边缘 flood-fill**（战斗背景草地 bg=(73,109,78)，tol 48 + 1px 膨胀吃绿边）；真草更亮更饱和所以留得住。
- 石头用户否掉了「两张灰图抠的 109 个」，换成**一张干净透明 GPT 图切 30 个**（更多、更柔和）。

### 1.2 VFX 逐帧切图——「等分切」翻车三连

VFX 目标：把透明动画大图切成 `frame0,1,2…` 的**统一画布**逐帧（每帧 tight-crop alpha bbox，再放进 `max(W,H)+pad` 的同尺寸画布里对齐）。

- **教训：等分网格必错**。AI 大图不是干净格：
  - `fireball_attack` 是**单行**、投射物间距不均、只占 x76–1373（等分 8 列会切掉 blob + 多出一空帧）。
  - `lightning` 是**上 6 帧 + 下 4 帧**（不是 5×2！），左对齐、间距不定。
- **可用算法**：内容感知**行**（`rowmask=alpha.any(1)`，合并最小 gap 到目标行数）→ 每行找**列缝**（gap-run）；缝数够就合并到目标数，**不够**（帧挨在一起，如宽爆炸）就把该行内容 x 跨度**等分**成需要的格；再 tight-bbox + 居中。
- 每行帧数：fireball [4,4,4,4]、icespike [4,4,4]、lightning [6,4]、fireball_attack 单行 8 簇。
- 预览：每帧贴深底 (30,30,38) → 自适应调色板 GIF（duration 90, disposal 2）+ 带标号的降采样 contact sheet。GIF 已 gitignore，用户后续在 Aseprite 里微调。

### 1.3 VFX 第二批 8 张 1024×1536——切错 → 用户抓 → 重切两轮

- **第一批尝试全错**：(a) 等分 H/R,W/C → 内容没填满画布（大边距）+ 行错位 → 抓到邻行的 blob（一帧两个火球）；(b) 内容 bbox 等分修好 5 张干净的，但 ice/lightning 列还是切进邻列尖刺；(c) 谷值吸附吸到空边距。
- **最终两类**：
  1. **干净网格**（fireattack2 4×3、fireball2 4×3、hellfire 5×3、icespike2/3 4×3）→ 把**整体内容 bbox** 等分成 R×C + tight-crop 居中。
  2. **不规则**（icespike4 =13[4,3,3,3]、icespike5 3×5、lightning2 =10[3,3,3,1]）→ 按行找内容带、**在列簇之间的 gap 中点切**、把宽于 1.6×中位宽的簇**自适应再切**、tight-crop 居中。
- **ROUND 2（07-04）用户又抓出 5 张还不对**（hellfire/icespike4/icespike5 没事）——**只有全尺寸 contact 才看得出**的缺陷：等分行把高效果切了 → 每帧混进**下一行顶部的一条淡带**；icespike3 最左有一条 baked-in 竖线；末行拉到图底抓到远处淡尾。**修法**：scipy `label` 清掉 `min(w,h)<=4 且 max>=40` 的细线；行=内容带；**内行按 gap 中点切**，首行顶/末行底**加 cap** 不去够远处淡内容；再列切 + 自适应。
- **lightning2 ROUND 3（07-04）**：用户说帧对不齐——我把帧**底对齐到了 baked-in 的帧号数字**（每格左上角有个淡蓝灰的 1..12，颜色不可分离）。修法：分析前先把每格左上 95×95 涂空（贴图源也涂），再切；`lightning2` 实为 3×4=12。
- **关键登记教训**：**着地/落地类效果（闪电、冰锥、爆炸）必须底对齐**（`oy=Hc-pad-h`），不能 bbox 居中，否则地面线逐帧漂移；contact sheet 画一条红色地面线 (`y=Hc-pad`) 验证。

## 四、战斗演出实测——四轮修 bug（本 session 重头，含近战/箭/闪电）

> **我跑不了 Unity**，全靠用户 Play 后反馈。每一条都写清**现象 → 根因 → 修法**。

### 4.1 ROUND 1（首轮实测）

1. **箭太大、没有弧线**：`RangedStrike` 改为按**长边**定射弹尺寸（0.4×目标身高），走**抛物线**（arc=dist*0.28, `y+=arc*4k(1-k)`），并让箭沿速度切线**转向**（箭头随上下扬俯）。
2. **⭐「素瑶点闪电术却放了个火球」根因**：`BuildUnit` 给**所有**单位的 `Stats` 第二参（MaxMp）传了 **0** → 素瑶（闪电 cost3 / 治疗 cost2）**永远 MP 不够**、放不出技能 → 玩家看到的是她的**普通法术攻击（火球）**。修法：`BuildUnit` 给任何有技能的单位 `MaxMp=20`。闪电这才真正放出来、播 lightningFrames。
3. **剑气不飞**（在目标脚下原地播）：`PlayVfx` 加 `travelFromX`（帧间把 x 从施法者脚下 lerp 到目标）+ `impactFrac`；剑气传 `impactFrac 0.82`（波到了才算命中）；所有技能 VFX 改**贴地**（pivot 0.5,0）。
4. **⭐近战不过去打**（原地挥空）：`Play()` 近战分支原来走到 `-strikeX`（左中，够不到右侧防守方）。改成走到 `meleeApproachX = spawnX - half*0.16`（**紧贴右侧防守方**）再挥再退；防守方反击走到 `-meleeApproachX`。
5. **音效**：脚步声由「走一次响一次」改成走路时**每 0.3s 重复**（DrainQueue Move 循环）；UI 点击音换 bank（去掉「怪怪的」kenney_click_1）。

### 4.2 ROUND 2（技能扩展 + 演出打磨）

1. **冰锥术**加给素瑶（切 `art/vfx/magic/icespike_a`→`Assets/Art/VFX/icespike`，`stage.icespikeFrames`，`BankFor "ice"`）。

3. **剑气第 1-2 帧拉长**（headHold：领先帧 ×2.4 时长，冲刺时就是这两帧成形的新月）。
4. **治疗改地图释放**：`PlayMapHeal` 把 heal 帧当**世界 SpriteRenderer** 播在友军身上、**alpha 0.6 半透明**、绿色 +N 飘字、不切战斗画面。
5. **火球改地图 AOE**：`Ability{MapCast,Aoe=Plus,range3}`；Domain `AoeFireballCommand`（十字 footprint 中心+上下左右、命中所有敌人、**只发 UnitDied**、public `List<Hit> Results`、**不发 UnitDamaged 所以不切演出**）；runner `SubmitAbilityCell` + `PlayMapAoe`（地图火球 VFX + 逐目标飘字 + `_displayHp` + drain 死亡）；输入**格子瞄准**。

## 六、图标「像素感」真相 + ⭐ 手改 meta 被 Unity 覆盖的坑

- 用户重导入后**图标仍像素**。放大对比：**源图和导入图本身都是干净高清**（`icon_skill.png` 剑刃平滑，不是像素画）→ 问题纯在**导入/显示**。
- **真根因 = 没 mipmap**：200px 图标在 HUD 里被 CanvasScaler 缩到 ~45px 显示，没有 mipmap 缩小就走样（细剑刃锯齿）→ 看着「像素」。压缩上轮已关，所以剩下的是缩放走样。
- **⭐ 大坑**：我上轮手改 `.png.meta` 里 `enableMipMap:0→1` **没生效**——查磁盘发现 `enableMipMap` 又变回 0（但同批改的 `mipMapsPreserveCoverage` 却留住了）。原因：**Unity 重导入时用它内存里的 importer 状态回写 .meta**，把我手改的关键字段 clobber 掉。
- **正确修法**：新 `IconImportFixer.cs`（菜单 `Tools/FantacyCentry/Fix UI Icon Import (HD)`）用 `TextureImporter` **由 Unity 权威设置**：`mipmapEnabled=true` + Bilinear + aniso 4 + Uncompressed + maxTextureSize 512 + `SaveAndReimport()`——不会被覆盖。**用户须跑一次该菜单**。
- 结论：源图只有 200px，若跑完还嫌不够 HD，就得重出 512px 图标。

## 七、`NullReferenceException` 崩溃修复（07-05）

- 报错：`BattleInputController.HandleClick (…) at BattleInputController.cs:305`，栈只到 `HandleClick` + `Update`。
- 定位：305 行是 `Unit clicked = state.UnitAt(cell);`。`BattleState.UnitAt` 本身**空安全**（遍历 `_units`，栈里也没有 `UnitAt` 帧）→ 说明是 **`state` 本身为 null**，即点击处理时 `runner.State` 是 null（战斗未就绪/已拆除，或场景重建后残留旧 runner）。
- 修法：`HandleClick` 顶部加 `if (state == null) return;`（系统边界的合理防御，输入可能在 State 存在前/后触发）。
- 提醒用户：若重建后仍偶发，多半是场景里残留了**两个 BattleRunner / 旧 InputController**，重跑一次 `Build Battle Scene`（会清理旧对象）。

## 九、经验教训（写给未来的自己）

1. **AI 大图从不是干净网格**：等分切必错。内容感知（行带→列缝→宽 cluster 自适应），且**全尺寸看 contact sheet**——缩略图藏得住「一帧两个火球」的双影，用户就是这么抓出来的。
2. **数据层的 0 会静默杀死整条链**：`MaxMp=0` 让法师永远放不出技能、表现成「点 A 放了 B」。看到「行为对不上选择」，先怀疑上游资源/属性被默认成 0。
3. **手改 `.meta` 靠不住**：Unity 会用内存 importer 状态回写覆盖。改导入设置一律走 `TextureImporter` 编辑器脚本让 Unity 自己写。
4. **区分「压缩伪影」和「缩放走样」**：块状=压缩（关 textureCompression）；细边锯齿=缩小无 mipmap（开 mipmap）。两个都排查。
5. **登记基线要一致**：着地类 VFX 底对齐、近战要真的走到贴身、飘字/血条在**命中瞬间**结算——所有「时机/位置」问题先想清楚锚点。
6. **崩溃栈看最深帧 + 被调方法是否空安全**：`UnitAt` 空安全 + 栈无其帧 → 是 `state` 本身 null，而非内部。系统边界（输入)加空守卫。
