## 一、背景：为什么走 HD 装饰路线（§8）

战斗 UX 早先想用一整套**像素 UI kit**（§1–§7）。06-19 定下大原则后改道：**uGUI 拥有布局 + 文字 + 血条，美术只做「装饰底」**（美术与布局**解耦**）。所以不需要整套像素 kit，只要 3–4 张 **HD 平滑金框**垫在 uGUI 背后即可。

- 像素件：Point 滤波、无 mipmap、`Assets/Art/UI/`
- HD 件：**Bilinear + 无 mipmap + 不压缩**、`Assets/Art/UI/hd/`

解耦的好处在这两天反复验证：**美术只要金框，文字数字全归 uGUI**，对齐就不会再像以前那样「猜 sprite 里烤死的槽位」而永远对不齐。

### 2.3 导入规则（关键三连）

`PixelArtImportPostprocessor.cs` 新增 `UIHdRoot="Assets/Art/UI/hd"` 分支，**早返回**：
- **Bilinear**（平滑非像素）
- **不压缩**（非 POT 1722×458 没法 BC 压，且压缩会糊金）
- **mipmap = 关**（⚠️ 关键坑：mip averaging 会把细金丝和深 navy 平均 → 金变暗绿。UI 固定尺寸不需要 mip）

### 2.4 ⚠️ 根因 bug：每帧 tint 把金框染暗

用户报「面板还是半透明 + 金色发暗绿」。查到 `UpdateInfoPanel` 每帧执行 `_infoBorderImg.color = team`（team 是蓝/红、alpha 0.95）→ 把金框染成蓝（暗/绿）+ 0.95 alpha（透出地图）。

**修**：`if (panelUnitInfo == null) _infoBorderImg.color = team;`——HD 金框保持白色不透明，只有程序化 fallback 才染色。Python composite-over-yellow 验证 PNG 本身确实不透明。

> 教训（这两天第二次中招）：**美术看着不对、但 PNG 已验证正确 → 去查运行时改 `Image.color/alpha` 的代码，别怀疑资源。**

### 2.5 清晰度修正（Lanczos 预缩 + UnsharpMask）

金丝在 Play 里发糊、Inspector 里却锐。根因：运行时 bilinear 把 1722px 缩到 ~780px 面板（@1080p）= 2.2x 欠采样 → 糊。修：Python 预缩到 1280×340（LANCZOS，分离 alpha 防边缘透明色渗进金）+ UnsharpMask(1.4/140/2)。1280 ≈ 面板 1.6x，留 4K 余量，运行时只做 1280→780。

### 2.6 立绘 v1 → v2

陆离立绘从 `art/undecided_art/profilepics/LuLi_007_00001_.png`（灰底）抠：PIL floodfill 多边种子 thresh 38 只去边缘连通灰（内部盔甲灰安全）+ GaussianBlur 羽化 → 裁半身胸像。v2 改成**无硬框胸像**，放在左金角**右侧**（不挡角花），顶部探出金边（梦战那种脸冒出来）。

## 三、06-21 上半场：横幅抠图血战（最终被 GPT-Image 终结）

> 这半天基本在和「**画出来的背景永远抠不干净**」死磕。最后的结论直接催生了下半场的突破。

### 3.1 品红抠图的正确姿势：UNMIX матте

硬阈值 + 腐蚀会在抗锯齿边留**粉色光晕**。正解 = **unmix 抠像**：
- 分离量 `m = min(R,B) − G`：navy m≈−22、gold m≈−100、品红 bg m≈240，干净分开
- 软 alpha = `clip((hi−m)/(hi−lo))`，lo=55 hi=140
- unmix 还原真边缘色 `FG = (observed − (1−alpha)·BG) / alpha`（BG=品红中位 ≈243,0,242）
- despill + 1px 腐蚀 + 保留最大连通块（甩掉漂浮的水印星，`scipy.ndimage.label`，scipy 本次装进 .venv）

### 3.2 一张蓝图出蓝/红两色

别生成两次。**只**对强蓝主导像素 `(B>R+28)&(B>G+20)` → HSV hue=2（+饱和）。这样**金（R 主导）和银高光（近中性）都不动**。早先失败：(a) 亮度-crimson 映射把金变灰；(b) HSV hue 区间 mask 误伤冷调金高光。

### 3.3 三张源图全被毙

- 第 1、2 张：源里**烤死了玫瑰金 pinstripe**（B≈G，粉不是黄），能修一半但用户嫌糙
- 第 3 张（棋盘格背景 `ikadnrikadnrikad`）：背景是**画出来的棋盘格**（白 255 + 灰 ~201，**不是真 alpha**）。按 `min>150 & sat<45` + 只留触边连通块抠。但用户看出：**圆角被腐蚀削平了 + 残留白格点**。

### 3.4 关键领悟

**模型「画」背景（品红 OR 棋盘格）永远做不到逐像素干净**——边缘必有过渡/残留，我再怎么抠都是补救。按钮三态勉强收了（`79xuvx` 源），横幅卡死。

用户一句点醒：「**PixelLab 的小人是真透明 PNG，就干净很多**」「你查一下 GPT-Image 能不能直接出透明 PNG」。

## 四、06-21 下半场：**GPT-Image 真透明 PNG 突破** 🎯

### 4.1 查证（官方文档，2026-06 拉的）

| 模型 | `background="transparent"` |
|---|---|
| `gpt-image-1` / `1-mini` / `1.5` | ✅ 支持 |
| `gpt-image-2`（最新） | ❌ 不支持（文档明说） |

**真 alpha**（像 PixelLab），不是画背景再抠 → 没毛边、没染色、没残格。**用 `gpt-image-1`。**

### 4.2 给用户讲清的两件事

1. **ChatGPT Plus/Pro ≠ API**：Plus 是网页订阅，**不送 API 额度**；真透明只能走 **API**（`platform.openai.com`，按量付费）。
2. **流程**：充值（最低 $5，用户充了 $10）→ 组织验证（GPT-Image 系列要求，用户已 Verified）→ 生成 API key。high 质量 1536×1024 ≈ **$0.25/张**。

### 4.3 新工具 `scripts/gpt_transparent_image_generation.py`

- 默认 `gpt-image-1` + `background="transparent"`
- key 从 `openaikey.txt`（repo 根，**已加 .gitignore，绝不进库**）或 `OPENAI_API_KEY` 读
- 预设 panel / banner-blue / banner-red / button / custom
- **`--ref <img>`（可重复）= init / 风格锚**，走 edits 端点
- 输出默认 `art/undecided_art/ui/hd/gptimage/`（无 .meta，先审）

### 4.4 ⚠️ 坑：第一张 panel 偏方——我没用原始 prompt

第一次跑 `panel` 预设，用的是我在脚本里**凭记忆默写**的 prompt，不是 §8.2 原文。两处差异：

| | §8.2 原始 | 我写的 | 性质 |
|---|---|---|---|
| 比例 | `roughly **4:1**` | `roughly **3:2**` | ❌ 错（手滑默写） |
| 背景 | 品红 #FF00FF | transparent | ✅ 必须改 |

`4:1→3:2` 让面板出成偏方胖款。**教训：写预设要直接粘原文再只改背景，别凭记忆重述。**

### 4.5 `--ref` init image 救场

喂**老 panel** 当 `--ref` → 模型照着真实 ~3:1 比例 + 细金角花重出一张**干净透明**版。一次就对。

### 4.6 又一坑：模型不居中，左边被画布切

第一张 ref 版左边金角顶到画布 x=0 被切。prompt 加「**居中 + 四周留白 + 不许碰/切任何边**」重出 → x 从 81 开始，四边完整。

### 4.7 入库

裁到 alpha bbox = **1374×786（~1.75:1，纯金框 + 透明中心）**：
- 生产：`Assets/Art/UI/hd/panel_info_frame.png`
- 老件备份：`art/ui/hd/panel_info_frame_OLD_backup.png`
- 源备份：`art/ui/hd/panel/panel_info_frame.png`

**决策**：其余 UI（banner/button/tile）都走这条路（`gpt-image-1` 透明 + 这张当 `--ref`）。

## 六、本两日「坑」清单（备忘）

1. **画出来的背景永远抠不干净**（品红 OR 棋盘格）。真 alpha（GPT-Image API / PixelLab）才是根治。
2. **gpt-image-2 不支持透明背景**，要用 `gpt-image-1` 系列。
