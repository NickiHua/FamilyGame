## 二、连续大图切片实验 —— 失败，回到单张

- 实验：让 **GPT-Image 生成一张连续纹理大图**，再程序切成 64×64 的块，期望「一次出多个 variant」。
- 结论：**接缝特别明显**、**像素感很差**。一张大图按格切，块与块边界对不上、量化也不和谐。
- 决策：放弃大图切片，**回到「单张 tile 出图 + 修」**。和 06-26 的「下采样定标准」一致，variant 靠单张轻量改色而非整图切。

## 三、Tilemap 调色板诡异 bug：tile 加进去选不中

- 把 tile 加到 Palette 后**在面板里选不中**（不知原因，疑似调色板缓存/资产状态坏掉）。

## 四、画完 Stage1 后的对齐/显示连环坑（已全部 code 修复）

新画的 Tilemap 命名 Stage1，导出走 `Tools/Map/Export Stage1 JSON`。期间四个坑：

1. **schema 不匹配**：导出器写 `grid_w/grid_h`，旧 `MapGrid` 只认 `grid_size` → 解析失败。MapGrid 改为支持 grid_w/grid_h（+ origin），新增 Width/Height/Origin，Size=Width 兼容。
2. **整图世界偏移**：用户为避免冲突把 tilemap 拖到非原点（-25,5），逻辑/gizmos/人物都假设 cell=world(0,0) → gizmos 与人物飘到一边、人物生成在地图外。引入单一 WorldOrigin：MapExporter 导出时**自动把 tilemap 吸附到底左格=世界(0,0)**，origin 写进 json；GridMover/RangeOverlay/BattleInputController 加 worldOrigin，gizmos/相机/平移/spawn 全部偏移。规矩：**重画后只跑 Export→Build，绝不手拖 XY**。
3. **人物被地图盖住只剩血条**：YSort 把人物 order 设 -footY*100（负），新 Tilemap order=0 盖住。`Build Battle Scene` 自动把所有 TilemapRenderer order 压 -5000，人物(-1500)/范围格(-4000)都在地图上。
4. **gizmos 差半格**：Tile Anchor=0.5，格中心在世界 +0.5，逻辑用整数(角)→错半格。origin 设 **0.5/0.5** 对齐格中心。约定：**tile anchor 0.5 ↔ origin 0.5 配套**，保持默认不动。
