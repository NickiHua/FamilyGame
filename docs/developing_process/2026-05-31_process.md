**Attack 自定义 prompt**：
```
Character raises sword from idle stance up and back over shoulder, 
winds up, then swings down in horizontal slash across body, 
body weight shifts forward with the swing, returns to idle. 
Feet stay planted, no spinning, no jumping. 
Sword visible every frame.
```

**Reactions 教训**：PixelLab 的 Taking Punch 预设按"现实主义"轻反应做的，48px 尺寸下几乎看不出。**结论**：sprite 保留，进 Unity 后用 闪白 shader + 后退 tween + 镜头震动 + 飘字 组合拳补强（这是 FE/Langrisser 的标准做法）。

## 四、共性问题与教训

### 4.1 PixelLab 侧视图丢武器（系统性问题）

**症状**：
- 苏瑶 EAST/WEST：兜帽自动戴上，法杖完全消失，变成"修女/幽灵"造型
- 帝国兵 EAST：盾牌占满身体挡住斧
- 帝国兵 WEST：斧和盾全部消失，变成"绿衣行人"

**根因**（推测）：
- AI 在侧视图下倾向把"远手"（背对镜头那只）的武器画丢
- 训练数据中侧视图常将另一只手放身后，被遮挡是常态
- 兜帽袍 + 侧视女性 → 训练数据里"修女/幽灵"语义压过"持杖法师"
- 这是**模型限制**，不是 prompt 写得不够

**对游戏的影响**：严重。SRPG 单位频繁东西向移动，玩家一眼就察觉"角色变了"。

### 4.2 扩散模型 prompt 写作教训

| 教训 | 例子 |
|---|---|
| 否定词不可靠 | "no horns" 弱，AI 仍画出角 |
| 用正向替换否定 | ✅ "plain steel kettle hat helmet" 替代 "no horns" |
| 文化锚定强于装饰词 | ✅ `european medieval` + `classic western` 比单纯描述配色更稳 |
| 配色与文化绑定 | crimson + horned → 日式；teal/brass → 西式 |
| 重复关键词增加权重 | "axe and shield visible every frame" 比只在开头说一次更稳 |

### 4.3 Custom V3 动画 prompt 模式（已验证有效）

**结构**：
```
[一句话总结动作]
F1-N 逐帧描述（描述身体/武器关键位置）
[硬护栏：feet planted, no spinning, no jumping]
[武器约束：weapon visible every frame, never dropped]
[末尾：风格词]
```

**字符限制**：1000 char 上限，建议 ~600 留余量。压缩技巧：
- 完整句变短语（"he lifts" → "lifts"）
- "Frame 1" → "F1"
- 同类禁令合并一行
- 最强形容词保留，弱的删

### 4.4 动画分类性能（已验证）

| 类型 | 通过率 | 备注 |
|---|---|---|
| 预设 Idle | 高 | 全方向稳定 |
| 预设 Walking | 高 | SOUTH/NORTH 优秀，E/W 受 rotation 质量影响 |
| Custom V3 Attack | 高 | 详细分镜 prompt 后稳定 |
| 预设 Reactions (Taking Punch) | 中 | 反应太轻，需 Unity 补强 |

### 4.5 动画与 Rotation 的依赖

**关键发现**：所有动画姿势从 rotation 模板派生。**rotation 错 → 所有方向的所有动画都错**。所以应该：
1. 先把 rotation 全部生成验收
2. 再做动画
3. 如果某方向 rotation 不行，先修 rotation，再重做该方向所有动画

## 六、下一步建议（最终结论）

### 建议路线（分两步，按顺序）

**Step 1：修 EAST rotation + 重做侧视动画（必做）**

砍掉 WEST 全部文件（Unity flipX 复用 EAST 即可，spec 已定）。重点修 EAST：

**苏瑶 EAST 强化 prompt 增量**：
```
side view from her left, hood down showing face and silver-blue hair, 
gold staff with large glowing blue orb held vertically in her right hand fully visible beside her body, 
staff and orb must be clearly visible, not hidden behind body
```

**帝国兵 EAST 强化 prompt 增量**：
```
side view from his left, 
battle axe held in right hand visible at side, 
round shield held in left hand in front of body but not covering the face or axe, 
both axe and shield clearly visible, axe head visible above shield rim
```

**测试方法**：每个先跑 1 张 rotation，看效果再决定后续。

**Step 1 fallback（如果 prompt 修不好）**：
- **B1（推荐）**：Aseprite 人工补武器（5-10 分钟/方向）
- **B2**：PixelLab Inpainting 工具框选手部，强制加武器
- **B3**：游戏内补偿（选中时武器高亮 + 装备图标 + 飘字反馈）

**Step 2：进 Unity（EAST 修好后）**

不再延后。Unity 阶段会暴露更多技术未知（PPU、animator、tile 对齐、闪白 shader），这些必须在 demo 完成前验证。

### 决策回顾

| 时间点 | 决策 | 是否仍然成立 |
|---|---|---|
| 早些时候 | "直接进 Unity，E/W 瑕疵在游戏里看不出来" | ❌ 收回，逐张审查后发现问题严重 |
| 现在 | "先修 EAST，再进 Unity" | ✅ 当前结论 |

### 长期教训

- **不要在没看图的情况下下定论**（"top-down 看不出来"是脑补，不是观察）
- **rotation 是基础设施**，rotation 不验收完，做动画就是在错误基础上叠加
- **PixelLab 侧视图丢武器是已知模型限制**，不是 prompt 问题，prompt 只能缓解
- **小型团队应预留 Aseprite 人工修补时间**，全 AI 生成不现实
