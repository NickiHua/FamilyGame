## 一、Aseprite 操作速查

### 修图时取色/填色颜色对不上（最常踩）
**原因**：铅笔默认 Ink 模式 = *Simple Ink*（alpha 混合）。如果目标像素是半透明的，画上去会和底下混合，颜色就变了。

**解法**：
1. 选铅笔 `B`
2. 顶部工具栏 **Ink** 下拉 → 改成 **`Copy Color+Alpha`**（完全覆盖像素的 RGB 和 alpha，不混合）
3. 画笔 **Opacity 拉满 100%**，关掉 Dithering / 抗锯齿
4. 修完记得把 Ink 改回去

### 让背景变黑（三种「背景」别搞混）
- **给图加黑底**（导出的图里有黑）：新建图层拖到最底 → 前景色黑 → `Ctrl+A` 全选 → `Backspace` 填充。有 Background 图层的话直接选中它填黑。
- **透明棋盘格看着黑**（只改显示，不影响导出）：`Edit → Preferences → Background`，把 checkerboard 两个颜色都改黑。
- **软件周围灰区变黑**：`Edit → Preferences → Theme` 换深色主题。

### 精确搬像素（保证颜色一致）
矩形选框 `M` 框住源像素 → `Ctrl+C` → `Ctrl+V` → 拖到目标。复制的是原始 RGBA，绝对一致，不走混合。

### 结论：Aseprite 不适合修立绘的头发边缘
Aseprite 是**像素编辑器**，一格格改半透明发丝极慢。头发/软边缘用 Photopea/PS 的 matte 处理更快。

## 二、Photopea 操作速查

### 找不到 Matting / 去白边
- Photoshop 有 `Layer → Matting → {Defringe / Remove White Matte / Remove Black Matte}` 三个命令。
- **Photopea 只给了 `Layer → Defringe`**（在 Layer 菜单**最下面**），而且是**单点固定 1px**、没有输入框，**没有 Remove White Matte**。
- 前提：图层必须**已解锁**（双击右侧图层面板解锁 Background），否则 Defringe 灰掉。

### Defringe 的实际效果 & 局限
- 点一下往内啃 1px（用旁边实心色替换边缘色），不够就**多点几次**。
- **对细发丝/白点无效**：它只处理「1px 边界」，AI 抠图那种满头发丝的半透明白点它救不了 → 用户这张点几次 Defringe 仍然不行。

### 想要纯二值边缘（没有半透明过渡）
下次用套索/魔棒前，工具栏**关掉 Anti-alias**，剪出来就是硬边 0/255。（但边缘颜色仍会被旧背景污染，见原理三。）

### 手动啃边（任何版本都能用，替代 Defringe）
1. 右侧图层面板 **Ctrl+点缩略图** → 载入不透明选区
2. `Select → Modify → Contract` 收缩 1px
3. `Select → Inverse` 反选
4. `Delete` → 删掉最外圈污染像素

### 真正治白边的方法（Photopea 做不了 → 用脚本或 PS）
Photopea 没有 Remove White Matte。**去白底**要么用 Python 脚本（公式见原理三），要么用 **PS 的 Select & Mask → Decontaminate Colors**（约 5-15 分钟/立绘，手动金标准）。

## 三、原理（为什么会这样）

### alpha ≠ 颜色
- RGB（24-bit）已经能显示**全部 1670 万种颜色**，alpha 是**第 4 通道，只管透明度**，跟颜色无关。
- **PNG vs JPG 的区别不是颜色多少**（一样）：PNG = 无损 + 支持 alpha；JPG = 有损 + 不支持 alpha。
- 只有**需要透明**（立绘抠出来贴背景）才用 alpha。

### 抠图为什么会凭空出现 alpha
JPG 打开是**锁定的不透明 Background 图层**。套索/removebg 要表示「这里没东西」，唯一办法就是把这些像素 **alpha 设 0** → Photopea 自动解锁图层 + 加 alpha 通道。**alpha 是「制造透明」这个动作创建出来的**，不是 JPG 带来的。

### 白边 / 毛边（黑底上的白色 halo）
- 从**白/浅色背景**抠图，边缘半透明像素的 RGB **混进了白色**。
- 放白底看不出来，放**黑底就露出一圈白边**。跟 anti-alias 开不开无关，**根源是颜色被背景污染**。
- **去白底公式**（反推真实颜色，假设边缘是叠在白 255 上合成的）：

  $$\text{真实色} = \frac{\text{观察色} - (1-\alpha)\times 255}{\alpha}$$

  能保留平滑边缘同时干掉白边。这就是 PS「Remove White Matte / Decontaminate」在做的事，Photopea 没这功能。

### alpha 二值化（让立绘好修图）
`alpha ≥ 阈值 → 255，否则 → 0`。身体变纯实心（取色填色不再混合），外部保持透明。阈值：低(90-128)边缘胖/留细节；高(200-230)边缘瘦/去毛边但吃掉剑尖发丝。luli 实测各阈值都约 52% 不透明，差别只在 1px 轮廓。

## 四、removebg vs Photopea 套索（本 session 的核心比较）

| | Photopea 套索 | remove.bg |
|---|---|---|
| 边界怎么定 | 你**手画**的几何硬边 | **AI 分割**预测前景 |
| 边缘 | anti-alias 抹平（一圈半透明） | **alpha matting**（满发丝的**大量渐变半透明**） |
| 懂画面内容 | 否 | 是（知道哪里是头发） |
| 去污染 | 不做 | 尝试做，不完美 |

**共同点**：两者都产生**带 alpha、被背景色污染的边缘**，都要后期清理。
**关键区别**：removebg 半透明像素**多得多** → 对它「砍掉 alpha<230」再二值化特别有效；也正因如此它在 Aseprite 里更难修。

## 五、结论 / 待办

- **原则（用户强调）**：AI 美术做到 95%，**最后 5%（一致性 + 干净边缘）最耗时**；要卖的游戏**不能用 removebg 直出**的边缘质量。
- **luli 立绘 3 版对比**：① 手动 Aseprite 修过的最干净（可用）> ② rembg + alpha<230 二值化（尚有残留）>> ③ removebg 直出（白边最脏）。
- **修边正确顺序**：removebg/套索抠图 → **Photopea/PS 去白底 + 手动精修** →（可选）二值化 → 进 Aseprite 补个别像素。

### 6.5 胸像下沿对齐 bug 修复
**现象**：面板里 6 个胸像**下沿高度不一**，lingshuang / 斧兵 / 队长明显上浮。

**根因**：胸像源图其实都对（都 300 高、内容贴底 `bot=299`）。问题是面板 `PortraitArt` 用了 `preserveAspect=true`——Unity 的 preserveAspect **垂直居中**；面板窗口宽高比按第一个（LuLi=1.17）算成固定值，比它更宽的胸像「适配宽度→高度变矮→上下留白居中」，于是下沿浮起来：

| 角色 | aspect | 修前 |
|---|---|---|
| luli 1.17 / suyao 0.94 / archer 1.15 | ≤1.17 | 贴底 ✅ |
| lingshuang 1.357 | >1.17 | 上浮 ~18px ❌ |
| axesoldier 1.380 | >1.17 | 上浮 ~20px ❌ |
| captain 1.310 | >1.17 | 上浮 ~14px ❌ |

**修法**：`PortraitArt` 锚点改成**底部居中**（pivot `(0.5,0)`），`preserveAspect=false`，在填充时按各自比例手动算适配尺寸（比窗口宽的适配宽度、否则适配高度），底边钉到窗口底部。缩放大小和原来 preserveAspect 一致（不会戳进右边文字），只是从「居中」变「贴底」→ 6 个下沿落在同一条基线。
