## 一、开场：GPT-Image 机制的三个研究结论（都写进 memory + 文档）

用户先抛了两个问题，顺带把 GPT-Image 的坑摸清了：

1. **「prompt 里加『输出透明背景』人物脸就崩」** —— 复现 + A/B 验证后确认这是**双重坑**：
   - **API 参数** `background="transparent"`：不但降画质，还会把**白/浅色区域打穿成洞**（苏瑶白袍的肩膀直接透明了）。
   - **prompt 文本**里写 "transparent background"：直接**毁脸**。
   - 结论 → **一律 OPAQUE 生成，事后再抠**。concept/立绘/胸像全用 `--background opaque`。
2. **「GPT 1.5 的 size 跟收费有关系么？还是只跟 quality？」** —— 查官方文档定死：
   - 计费是**按输出图 token**，**size 和 quality 都影响**；总价 = 输出 token + 输入文本 token + 输入图 token（`--ref` edits 会多几分钱）。
   - 1/1.5 **只允许固定 size**：`1024x1024` / `1536x1024`（横）/ `1024x1536`（竖）+ auto，**没有 1536x1536、不能任意尺寸**（只有 gpt-image-2 能任意 size，但它不支持透明）。
   - `gpt-image-1.5` **又好又便宜**（比 1 新一代、更省）：high 下 `1024²=$0.133`、`1536×1024=$0.20`；gpt-image-1 是 `$0.167 / $0.25`（用户之前看到的 ~$0.25 其实是 gpt-image-1）。默认模型切到 **gpt-image-1.5**。
3. **Images API 不会改写 prompt** —— 官方文档确认：`images.generate/edit`（1/1.5）**逐字使用** prompt，**不扩写**。改写只发生在 Responses API 的 image_generation tool、ChatGPT 网页、和 DALL·E-3（`revised_prompt` 是 dall-e-3 专属）。所以 → **写英文、写具体**，抽象的质量形容词是白扔 token。

> 附带结论：**1:1 + low quality 裁头最狠**，且此时 margin/「留白」文本经常被忽略。真正解法 = 用非正方 size（立绘用竖 1024x1536）+ medium/high quality + 硬 framing 约束块（"head/hair/feet fully inside，装不下就 ZOOM OUT"）。

## 二、生成方法的演进：三步收敛到「合并大图 combined sheet」

方法迭代了三代，最后定在合并大图：

1. **concept → 分开生成立绘 + 分开生成胸像**：胸像**逐张生成会漂**（每次生成衣服/发型/构图都不一样）。
2. **胸像改 2×2 一次出**：`portrait-busts` 预设 = 一张 1024x1536、2×2 四表情（TL 中性 / TR 笑 / BL 悲 / BR 怒），硬约束「**只有表情变，衣服/发型/构图完全一致，腰以上**」→ 4 张胸像完全一致。
3. **最终 = 合并大图（combined sheet，本 session 定的最佳法）**：**一张 1536x1024 opaque 大图** = 右边全身立绘 + 左边一列表情胸像 → 立绘和胸像**共享完全相同的衣服**（一次生成、零漂移）。
   - 布局：`_PORTRAIT_SHEET_4`（友军，左侧 4 竖排胸像，表情用 `--busts "e1|e2|e3|e4"` 传）+ `_PORTRAIT_SHEET_1`（敌军，1 张中性胸像）。
   - 每角色一个 `<char>-sheet` 预设。
   - 一个切图器把大图拆开：**列投影找竖向空隙** → 右区 rembg = 立绘，左区按行切（友军 4 / 敌军 1）rembg = 胸像。

- **敌军脸/眼定调（用户决策）**：杂兵（弓手/斧兵）= **冷、小、窄眼**、凶相、露脸不打阴影，**不给英雄大眼**；队长 = **闭面甲、全无脸**（无眼）。全体敌军 = 帝国 **深青绿 + 钢 + 黄铜 + 棕** 配色（对比友军的 白/金/银/粉/蓝）。敌军只要立绘 + 1 中性胸像（不做 4 表情，对话里不 emote）。
- **蓝眼**：HIGH 质量下自然稳定，不靠 "not green" 这种 prompt hack（用户原话：**眼睛是抽卡，重摇不是塞 prompt**）。
- **武器全保住**：这轮 HIGH/medium 大图里 staff/spear/sword/bow/axe+shield/captain sword+shield **全部原生 rembg 就抠出来了**，没用补丁（对比：早前苏瑶低细节的细法杖才需要 flood 补丁）。

## 三、抠图方法：按资产类型分流（立绘软 / 胸像硬）

**核心 = OPAQUE 生成 + rembg 抠**（rembg 2.0.76 `isnet-anime` 模型，动漫训练的语义分割）。**绝不能对白袍/银甲做颜色键/flood-fill**（白≈浅灰底，会把袍子/盔甲吃掉）。抠图方法分两类，是踩出来的：

- **立绘 = rembg 软 alpha**：软抠边适合武器/头发；HIGH 大图上武器全过。
- **胸像 = rembg + 硬阈值 alpha**（**不是 flood-fill！**）。三步教训：
  1. 纯 rembg 胸像**底部「糊」/软** = rembg 在切口处的**羽化 alpha**。
  2. 试 flood-fill 求脆边 → **flood 吃了白袍/银甲**（白/银≈浅灰底，且胸像构图里衣服贴到左右边框 → 与边框连通 → 被删）。
  3. **最终修法** = rembg（语义、保住白）**再硬化它的 alpha**（`alpha>=128→255 否则 0`）+ 1px 腐蚀 → **脆边且白色安全**。
- **友军 4 胸像切分** = 左列内容 y 范围**等分四份**（稳；早前用「找间隙」把凌霜切错过一次）。
- **敌军胸像裁切比例**（保留顶部）：captain 0.50（胸）、axeman 0.80、archer 从 0.68 调到 0.82（太紧）。
- 切图器都是**一次性临时脚本**（跑完删）——用户拍板**不做统一的抠图脚本**（「每个立绘都不一样，没法统一」）。

## 八、经验教训（写给未来的自己）

1. **看不到的图不要假装看到**：`view_image` 返回「budget exceeded / Image omitted」= 我没看到。谎称看到并描述会误导用户、砸信任。只描述真正加载成功的图。
2. **透明背景是双坑**：`background="transparent"` 参数降质 + 打穿白色；prompt 文本写 "transparent background" 毁脸。→ **一律 opaque 生成，事后 rembg 抠**。
3. **白/银主体绝不 flood-fill**：白袍/银甲≈浅灰底，flood 会连边框一起吃掉主体。要脆边就 **rembg 语义抠 + 硬阈值 alpha**（保白又脆）。
4. **一次生成 > 分次生成**：立绘/胸像分开生成必漂移。合并大图（右立绘 + 左胸像）一次出，衣服零漂移。
5. **Images API 逐字用 prompt、不扩写**：写英文、写具体，别指望它帮你扩写抽象词。
6. **本地免费的随便跑，花钱的先确认**：rembg/切图/渲染免费 → 直接迭代；GPT-Image 每次调用先跟用户确认。
7. **每张立绘单独抠**：没有万能脚本，用一次性临时脚本按图调参，验证后删。
