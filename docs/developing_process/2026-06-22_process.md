## 一、UI 出图 Pipeline 定型（本两天最大的方法论收获）

### 1.1 二级暂存 + 三级落地

```
art_undecided/ui/<type>/   ← AI 原始产出（无 .meta，先审，可能要返工）
        ↓ 审核通过
art/ui/<type>/             ← 批准的 master（仓库内长期保存的「事实母版」）
        ↓ 装机
Assets/Art/UI/<type>/      ← Unity 生产件（带 .meta：Bilinear / 9-slice / GUID）
```

`<type>` = `panel / button / icon / range / portrait …`。脚本 `install_icons.py` 就是按这条路把图标从暂存搬进 master + Assets，并自动处理 meta（旧名覆盖**保留 GUID**、新名按模板克隆 + 换 GUID + Bilinear）。

### 1.2 核心原则：**AI 只做一块标准底，数量/排列/状态/对齐全归 Python**

06-21 已悟「画出来的背景永远抠不干净」；这两天进一步定死了**生成策略**本身：

> **GPT-Image 是扩散模型，有强烈的「中心偏置」且不会数数。** 让它一排画 N 个等距物体时，会**系统性地把最左边那个切掉**、或**少画**（要 7 个出 6 个，要 3 个出 2.5 个）。prompt 写 "none cropped by any edge" 只能缓解，**不是硬约束**。
>
> **根治**：让 AI 只出**一个**标准件，**数量 / 排版 / 对齐 / 三态**全部用 Python 控制。按钮三态就是这么做的（见三）。

### 1.3 关于「光影感」与「好不好抠」的认知（和用户的几轮讨论，值得记）

- **烘焙光影 vs 纯色+引擎打光**：2D UI 业界主流是**把光影画死进贴图**（baked）。专业管线确实留**分层源文件**（albedo / highlight / shadow / AO 分层），但出货是压平的 PNG。小作坊**单层 PNG 完全够**，差距只在可编辑性，不在最终画质。我们的「选一张满意空框当母版 + Python 合成」就是轻量版图层管线。
- **位图必然像素化**：只要不是矢量，放大超过原生分辨率就出马赛克——这是物理规律不是 bug。关键是**原生分辨率 ≥ 屏幕显示尺寸的 1.5–2×**即可（图标 200px 源 → 屏上 48–64px 显示，是降采样，清晰有余）。
- **「框外淡色光晕」= Outer Glow / Bloom，是抠图的敌人**：带烘焙光影的图，亮部往轮廓外溢一圈**半透明渐变** → 阈值抠图要么留色边要么啃边。`erode`（腐蚀）是**解药**不是现象名。专业做法是**把发光单独放一层**（Screen/Add 混合，甚至丢引擎做 Bloom），本体保持**干净硬边剪影** → 好抠。这正是下面 Gemini 比 GPT 好抠的根因。

## 二、Gemini 12 图标：真正好抠的那套 🎯

### 2.1 为什么换 Gemini（GPT 透明 PNG 反而难抠）

GPT-Image 的透明底是**带羽化外发光的渐变 alpha**：边界半透明、糊，怎么抠都留毛边。
用户改用 **Gemini** 出了一张 **2×6 网格、12 个图标、画在不透明纯黑底上**的整图——**硬边 + 几乎无外溢** → 轮廓清爽，反而**比 GPT 的「现成透明 PNG」好抠得多**。

> 认知固化：**「好抠的资产 = 干净硬边 + 发光分离」**。AI 在不透明纯黑上画硬边图标，比让它直接出羽化透明 PNG 更可控。

### 2.2 抠图脚本 `scripts/cut_gemini_icons.py`

整图 1408×768，6 列 3 行（第 3 行是 Gemini 多画的重复，只用前两行 = 12 件）。先用「金量投影」量出网格列/行边界，再**逐格 flood-fill**：
- 黑背景 = `maxRGB < 60`；从「框内内沿一圈种子 + 外边框」连通域填充 → 标记为背景 → 透明
- **金框保留**（金色不在黑 mask 里）、**物体保留**、**物体内部暗部不会被误删**（被物体包住、不与背景连通）

一次抠干净，12 个透明 PNG + 品红底预览 montage 给用户审，无毛边、无破洞。

### 2.3 12 件命名 + 落地

第一行：空框 / 攻击(剑) / 技能(双剑) / 魔法(书) / 防御(盾) / 待机(沙漏)
第二行：道具(药水) / 移动(靴) / 变身(角) / 装备(戒指) / 状态(人) / 召唤(狼)

`install_icons.py` 装进 `art/ui/icon/`（master）+ `Assets/Art/UI/icons/`：**Bilinear**；6 个旧名覆盖保留 GUID，6 个新名克隆 attack 的 meta + 换 GUID。空框 `icon_frame_empty` 留作**模板母版**（以后新图标贴进它即可，统一性天然保证）。

> 实际用到指令面板的 prompt 已存进 `docs/prompts/ui-prompts.md` §8.8。

## 三、按钮三态：Python 派生 + 统一画布修「悬停金边缩水」

### 3.1 GPT 切左边 → 本地派生

GPT 出的三态横排被中心偏置切了左边。**解法**：取**中间那个完整按钮**当 base，Python 派生三态——
- Normal = base；Pressed = `adjust(0.68, 0.92)`（压暗、降饱和，做下沉感）
- Hover = `adjust(1.28, 1.12)`（提亮）+ **均匀蓝光**

### 3.2 ⚠️ bug：悬停时金边「变小」

第一版 hover 画布做大了（541×348）但金芯没放大 → 金框看着**比 normal 细一圈**；而且 v1 的辉光只从圆角漏出来、四角发蓝很怪。
**修**：三态**统一同一画布 541×348**，金芯**放在完全相同的位置**（offset 30,30），**只改亮度**；蓝光用 `binary_dilation + gaussian_filter` 做**整圈均匀外发光**叠在 hover 后面。9-slice border 三态统一 **100**（30px 边距 + 70 金边）→ 金框三态**像素级一致**，悬停只多一圈光、不缩水。

> 教训：**三态要保证金框几何完全一致，差异只能来自亮度/叠加层，不能来自画布尺寸。**

## 五、资源大清理 + 全面 HD 化

### 5.1 目录重组

- `hd/{panel, button_*, portraits}` 上移到顶层 `panels/ buttons/ portraits/`；`panel.png → char_panel.png`（以后还会有别的 panel）
- 删 `hd/`（含旧 banners）、`nameplates/`、`bars/`、顶层 `banners/`、`panels/` 里多余 4 张
- `range/select_frame.png` 换成 HD 版（四角金回纹 + 中空，比旧像素角块更像「选中光标」）
- 清理后 `Assets/Art/UI/` 只剩：`buttons / icons / panels / portraits / range`

### 5.2 ⚠️ 连带 bug：postprocessor 会把新 HD 件强制成 Point

`PixelArtImportPostprocessor` 原来靠 `hd/` 子目录识别 HD 件；HD 件移出 `hd/` 后，**所有 UI 落进了「像素分支」→ 重导入会被强制 Point 滤波 + 旧 border**（图标变锯齿、按钮/面板 9-slice 错）。
**修**：既然像素 UI 已弃用，把 postprocessor 的 UI 分支整体改成**平滑 HD**：Bilinear、buttons border 100、panels border 140。

### 5.3 回合横幅：从「图片」改成程序化色带

用户研究了别的 SRPG，决定不要特制 banner 图：改成**贯穿左右的红/蓝半透明色带 + 斜体 PLAYER/ENEMY PHASE**，纯 uGUI 代码画（`bandColorPlayer/Enemy`，`peakAlpha=0.6`）。顺手删掉了两套 banner 美术 + 接线。

## 六、角色指令面板（本两天的 UI 大件）

### 6.1 设计：复用 char_panel 拉成竖长条

参考火纹 / 梦战 / FFT：**梦战和 FFT 的指令面板与角色面板是同一个框**。所以**复用 char_panel**，9-slice 拉成右侧竖长条（金角固定、navy 中段拉伸），每行一个指令：**图标左 + 文字右 + 凹陷暗槽**。凹陷槽 = 白色 `RoundedSprite` 被 Button 的 state color 染：normal 暗、hover 暖金。

### 6.2 交互：菜单驱动 → 又改回直接鼠标驱动

先做了**菜单驱动**（MOVE/ATTACK/SKILL/MAGIC/ITEM/WAIT，点 MOVE 出蓝、点 ATTACK 出红、攻击只从当前格算）。用户实测后**改主意**：移动和普通攻击**直接鼠标驱动**手感更好——
> **最终交互**：选中单位 → 地图**直接出红蓝区**；点蓝格移动、点敌人攻击（可一键移动到位+攻击）；右键/Esc 撤销/取消。**指令面板只留 SKILL / MAGIC / ITEM / WAIT**（WAIT 可用，其余置灰留给以后）。

## 七、本两天「坑」清单（备忘）

1. **GPT-Image 中心偏置**：一排画 N 个会切最左/少画。根治 = AI 出一块标准件，数量/排版/三态归 Python。
2. **GPT 透明 PNG 反而难抠**（羽化渐变 alpha）；**Gemini 硬边纯黑底好抠**。好抠 = 干净硬边 + 发光分离。
3. **悬停金边缩水**：三态画布尺寸不一致导致。三态必须同画布、同金芯位置，只改亮度 + 叠光。
4. **PowerShell heredoc 把 agent 终端搞死**：`@'…'@ | python` 后台跑挂了 python 子进程，新终端全部卡空白、命令不落地，**重启 PC 才恢复**。教训：**别在 agent 终端用大 heredoc 管道喂 python，写成脚本文件再跑。**
5. **缺中文 TMP 字体**：指令标签中文渲染成豆腐块 + 刷 warning。**先用英文标签**清掉 warning；待导入中文字体再换回。
6. **UI 点击穿透**：点面板按钮那帧鼠标点击同时被当成点地图 → 取消了选中。修：`EventSystem.IsPointerOverGameObject()` 拦截。
7. **9-slice border 太小**：char_panel 四角花纹实测 ~130px，旧 border 110（甚至 48）< 花纹 → 一拉就一边长一边短。量准后**改 140**。
8. **信息面板横向拉长**：char_panel（比例 2.1）用 `Image.Type.Simple` 硬塞进 780×200（比例 3.9）→ 横向拉伸。改 **Sliced** 修复。
