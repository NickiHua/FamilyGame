## 0. 模型结论（这个 session 最重要的沉淀）

- **GPT-Image 整体明显好于 Seedream 5.0 lite**。lite 出的东西偏「整齐但通用」，细节/线条/氛围都比 GPT 弱一截。
- **GPT-Image ≈ Seedream 5.0 pro**（质量一个档），但 **Seedream 5.0 pro 更能 follow 指令**（尤其「保持原图只改一处」这种编辑类任务）。
- **Seedream 参考图（ref-image）一致性极强** —— 适合「同一角色/同一张图，只改动作/位置/加一个元素」的差分与微调；GPT 的 edits 也支持 ref，但改动幅度更大。
- **透明 vs 不透明**：GPT `background="transparent"` 会**明显掉质量**（脸糊、白/浅色区破洞、羽化边难抠）。**结论：一律不透明出图**，UI 硬边件用**纯黑底**（好抠、不污染边），立绘用**纯白底**。
- **抠图**：不透明黑底 → **flood-fill 抠**（不是简单 color-key）—— barrier = 非背景色，从图像边界泛洪背景色，enclosed 的孔洞 = 内部镂空。white-on-white / navy-on-navy 都能干净抠出来。
- **各资产实际用的模型（补图时照此对齐，别混模型免得画风漂移）**：
  - **状态图标 = Seedream（5.0 lite，纯黑底）** → **以后补新状态也用 Seedream**，拿 `status_seedream_black.png` 当 ref 续画。
  - 5 个角色天赋 cut-in + UX 框 + 对话框/爆闪 = **GPT（gpt-image-2 medium）**。
  - 立绘 = **GPT（gpt-image-1.5 high）**；立绘的差分/加疤类微调走 **Seedream ref-edit**。
- **模型型号踩坑**：
  - GPT：`gpt-image-2`（本 session 主力，medium/high）、立绘用 `gpt-image-1.5 high`（pipeline 默认）。`gpt-image-2` 支持任意尺寸、不支持透明（我们本来就不透明）。
  - Seedream：byteplus 端点该账号**未激活**（`ModelNotOpen`）→ 全部走 **volcengine 端点**（`doubao-seedream-5-0-260128` = lite，`doubao-seedream-5-0-pro-260628` = pro，读 `volcenginekey.txt`）。
  - Volcengine Seedream **尺寸下限 ≥ 3,686,400 px**（1024×1536 太小会报错；立绘用 1664×2496，方图用 2048×2048）。
  - **Seedream PRO 不支持 `sequential_image_generation` 字段** → 报 `InvalidParameter`。**修了两个脚本**：`byteplus_seedream_portrait.py`（`--sequential-image-generation omit` 可去掉该字段）和 `volcengine_seedream_ref_portrait.py`（model 含 "pro" 时自动省略）。这两处是**保留的正式改动**。

## 1. 状态图标（buff / debuff）

### 决策
- **框 = 细银边、小圆角方框、镂空**；一套**白色 glyph**同时服务 buff 和 debuff。
- **buff / debuff 靠底色区分**：镂空中心填**半透明蓝（buff）/ 半透明红（debuff）**，glyph 永远白色不变（用户明确：攻击 buff/debuff 就是同一把白剑，只有底色蓝/红不同）。
- 已生成**一组 10 个状态**：空框母版 + 攻击(剑) / 防御(盾) / 移动(翼靴) / 沉默(斜杠气泡) / 热血(火) / 中毒(骷髅滴) / 无法移动(链靴) / 无法行动(铐手) / 眩晕(星旋)。
- ⚠️ **状态图标是 Seedream 出的**（Seedream 5.0 lite，纯黑底），不是 GPT。**以后补新状态图标也要继续用 Seedream** —— 拿现有的黑底整图 `status_seedream_black.png` 当 ref、沿用同一套 prompt 续画，才能保证新图标的框/线条/风格与这 10 个一致（换 GPT 会画风漂移、对不上）。

### 出图（Seedream 5.0 lite，volcengine，纯黑底，`3072x1280`）
（先试过 GPT 透明/不透明、Seedream 白底 navy 底多版，最后定 **Seedream 纯黑底**做源 —— 黑底 flood 抠最干净）
> 注：实际跑的 status prompt **原文是英文**（经 orchestrator 传参，见附录 §A）；下面这段中文是等义说明。
```
生成一张 2x5 网格（正好 2 行 5 列，共 10 格）小方形状态图标。10 格共用同一个框：一圈细的、柔和的、
哑光的银色圆角方边，朴素无装饰。每个框中心是纯黑镂空，中间放一个扁平实心的纯白 glyph。10 格依次：
1 空框；2 竖直的剑（攻击）；3 骑士盾（防御）；4 带翼的靴（移动）；5 被斜杠划掉的对话气泡（沉默）；
6 升腾的火焰（热血）；7 滴落带小骷髅的毒液（中毒）；8 被锁链缠住的靴（无法移动）；9 一双被铐住的手
（无法行动）；10 三颗旋转的星星（眩晕）。每个 glyph 同样的纯白、同样简洁醒目的剪影风格、大小相近、
居中。整幅纯正黑底（RGB 0,0,0）填满画面和每个框内部——唯一的非黑元素就是细银框和白色 glyph。
高分辨率、平滑、抗锯齿，扁平均匀打光，不是像素风，无文字/数字/水印。
```

### 抠 + 上色（`scripts/_tmp_status_*` 临时脚本，flood-fill）
- **flood-fill 抠**：`content = max_channel > 55`（黑底上银框/白 glyph 都亮）；`binary_closing` 封 1px 缝；`fill_holes` 得到每个 tile 的实心六边... 方形；从图像边界泛洪黑背景 = outer；enclosed 孔洞 = 中心镂空 → 透明。
- **glyph 保持实心白的关键坑**：一个 tile 里有**两块 enclosed 白/黑区** —— 「tile 中心镂空」要染色，「glyph 自己的内部孔」要保持实心。**按面积区分**：10 个最大的 enclosed = tile 镂空（染蓝/红）；其余更小的 enclosed = glyph 内部 → 强制刷白+不透明。
- **蓝/红填充烤进 PNG**（不是运行时 tint）：中心镂空填**50% alpha 的竖向渐变**（上亮下深），buff 蓝 `[70,140,255]→[28,78,200]`、debuff 红 `[255,150,150]→[190,26,26]`；银框 + 白 glyph 不透明压在上面；框外全透明。用 `fill_holes` 让色填到银框**内沿之下**，消掉「色块和框之间的暗缝/缺角」。
- **毛边治理 = 母版框**：AI 出的 10 个框边缘各有细微毛边、不完全一致。用户提出「**做一个完美母版框 + 抠出每个 glyph 放进去**」。做法：**程序化画一个绝对干净的银色圆角方框（矢量级抗锯齿、无毛边、可 9-slice）**当母版，从黑底源里**只抠 glyph**（取每 tile 中央 74% 区域的亮像素、归一成纯白居中），合进同一个母版框 → 10 个框像素级一致。用户看过母版后拍板「就用这个框」。

## 3. 5 个角色天赋图（全 GPT-image-2 medium，opaque，`--ref` 角色立绘，1024×1024）

**统一风格**：深色逆光**剪影**（或半剪影，脸/表情适度受光）+ **放射高光背景**，颜色区分角色。之后**裁进六边形金框**。

> **头位置铁律**：构图上**头顶要留出空间**、别顶到画面上边 —— 因为裁进六边框时若头贴顶会顶到金框；
> 让 AI 在图里就留好上方留白，下游裁切才好做。

### 六边形镶嵌方法（复用于全部 5 张）
1. 从 `talent_gpt_medium.png`（2×2 金色六边 sheet）的**左上空框**抠出**空六边金框**：`content = alpha>0.5` → `fill_holes` 得 `hexfull`（实心六边=裁切形状）+ 金边 alpha + `inner_top`（金边内沿行号）。
2. **人物图裁进 `hexfull`**、**金框压在上面**（`alpha_composite`）→ 六边形把图裁掉、金边盖住接缝。
3. **top 对齐**：把人物图的**上沿对齐到金框内沿**（`inner_top`），头就不会顶框、上方留白露出来。
4. **每角色一组参数**（`zoom / top_crop_frac / x_shift / pad_frac`）微调：
   - LingShuang：`zoom 1.30`（放大到头→大腿），top 对齐。
   - LuLi：`zoom 1.15, top_crop 0.14`（LuLi 图头本来就没顶，先裁掉顶部多余蓝光再对齐）。
   - SuYao：`anchor=center, zoom 1.0, pad_frac 0.20`（物件居中；edge-扩展背景把六边填满、不露黑边，物件相对缩小留光晕）。
   - Skadi：`zoom 1.10, top_crop 0.25, x_shift 32`（人物+匕首往上提、往左靠、放大，让整把匕首露出、集中）。
   - Xiao：`zoom 1.10, pad_frac 0.13`（edge-扩展缩约 20%，让左边狼头 + 右边弓两头都进框）。

## 4. 立绘：Skadi + Xiao（走 portrait pipeline）

- **模型**：`gpt-image-1.5 high`（立绘美术上限、最可控）、**纯白底**（好抠、铁律 4 魔术棒选连通纯白）、`--ref art/portraits/luli/_opaque_source/luli_concept.png` 当**风格锚**（只借画风，反复强调「不同角色、别抄参考的性别/脸/配色」）。
- **只出一张完整全身立绘**（不走「右立绘+左四头像」sheet），平静表情。
- **统一性措辞**：明确「**少配饰、装饰极简、限定配色、融入全队美术、不繁复**」—— 否则 AI 会堆绑带/首饰/花纹。

### Skadi（帝国女将军/刺客：紫长发、红眼、黑刺客甲、红披风）
- v1 太像主角团正气脸 + 露脐露肩 → v2 改：**邪 villainess（但保清亮大眼、别缩成小眼 —— named boss 靠表情不靠小眼睛）** + **遮脐遮肩、性感靠贴身剪影+高开衩+长腿不靠露肤**。

consistent with the party; villainy via EXPRESSION - cold cruel arrogant, half-lidded disdainful
seductive gaze, one eyebrow raised, faint cruel mocking smirk; femme-fatale; NOT wholesome/heroic.
OUTFIT (sexy but COVERED): sleek form-fitting BLACK plate-and-leather that COVERS torso+stomach
(NO navel) and COVERS shoulders with pauldrons (NO bare shoulders); sexy via tight silhouette,
long legs, thigh-high boots, high side thigh-slit - NOT bare skin; deep CRIMSON-RED cape.
```

### Xiao（游侠/猎人：少年、棕发、绿眸、绿装+棕皮甲、脸侧两条前辫、背弓箭）
- **喂 2 张 ref**（GPT 支持多图）：`luli_concept`（风格）+ `xiao/rotations/south.png`（形象：前辫/肩甲/配色）→ 前辫、肩甲、结实体格一次到位。
- **一字疤是这个 session 的老大难**：用户要**左脸（画面右侧）眼下一道淡淡的单横线疤**。
  - **GPT 和 Seedream（lite/pro）都画不好**：默认画成**发红的小十字/两笔叉**（「不是十字疤」这种**否定词反而被当提示**照画十字；「伤疤/血」措辞 → 发红）。改成正面词「**一字疤**（「一」字形单横线、淡色近肤色）」也只把它掰横一点、仍偏红偏糙。
  - **结论**：小面疤 AI 不稳，**最可控是抠图那步在 Photopea/PS 手画一道淡线**；或 Seedream ref-edit（pro 更听话但仍不完美）。
```
BUILD: BOYISH teenager ~15-16, slender but WIRY and ATHLETIC (NOT frail/scrawny).
HAIR: tousled BROWN hair; TWO thin BRAIDED strands hanging at the FRONT, one beside EACH ear.
OUTFIT: green tunic + brown leather straps + a leather SHOULDER PAULDRON (like the sprite);
GEAR: wooden BOW (held at side OR slung on back) + a QUIVER on his back. NO cape.
