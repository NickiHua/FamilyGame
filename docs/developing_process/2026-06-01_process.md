## 二、方法变更（vs 5/31）

### 2.1 动画生成模式统一切换为 Custom V3

| 项目 | 5/31 | 6/1 |
|---|---|---|
| Idle/Walking/Reaction | 预设模板 | **全部 Custom V3 + 详细 prompt** |

| 视角约束 | 不写 | **明确写"3/4 view, not pure side"** |

### 2.2 "Keep First Frame" 设置带来的新现象

PixelLab 勾选"Keep first frame"后，文件命名是 `frame_000` 到 `frame_008`（9 个文件）：
- **F0** = rotation 参考帧（直接用 rotation 模板的那张图，纯侧视）
- **F1-F8** = 动画生成的 8 帧（3/4 view 动画姿势）

**新问题：F0 视角跳变**
- EAST/WEST 的 F0 是**纯侧视**（rotation 参考），F1-F8 是 **3/4 front view**（动画）
- 循环播放时 F8 → F0 会有明显"咔哒"视角跳
- NORTH/SOUTH 没这个问题（rotation 和动画视角一致）

**解决方案**：Unity 内 E/W 动画切片只用 F1-F8（8 帧循环），跳过 F0。N/S 可以用 F0-F8（9 帧）。

## 四、共性结论更新

### 4.1 5/31 的 §4.1 "侧视图丢武器系统性问题" — 重写

**5/31 结论**："这是模型限制，不是 prompt 问题"

**6/1 修正**：模型限制**可以通过 prompt 工程显著缓解到接近完美**。组合拳：

1. **强制 3/4 view**：prompt 写 `3/4 front view, not pure side profile`
2. **武器位置硬约束**：每个 F1-F8 分镜里**明确写武器在哪只手哪个位置**
3. **末尾重复武器约束**：`Weapon visible every frame, never dropped, never hidden`
4. **统一 8 帧 + Custom V3**：放弃预设模板（预设对侧视 + 持械角色容错差）
5. **配色与文化锚定不变**：5/31 的 prompt 教训继续生效

**结果**：12 套动画（3 角色 × 4 动画）中，**E/W 武器丢失/服装变形问题 100% 消失**。

### 4.2 新增问题：F0 视角跳变

| 项目 | 描述 |
|---|---|
| 现象 | E/W 动画 F0 是 rotation 参考（纯侧视），F1-F8 是 3/4 view 动画 |
| 原因 | "Keep first frame" 让 F0 用 rotation 模板原图，未参与动画生成 |
| 影响 | 循环播放有"咔哒"视角跳 |
| 解决 | Unity 切片只取 F1-F8（8 帧循环）。N/S 不受影响，可取 F0-F8 |
| 是否阻塞 | 否（Unity 配置即可，不需重生成） |

### 4.3 Reaction 偏弱依然是通病（但不是 blocker）

| 现象 | 结论 |
|---|---|
| 陆离 E/W 拳击护头 | 接受 + Unity 补强 |
| 苏瑶 全 4 方向 | ✅ 解决（N/S 戏剧，E/W 防御姿）|
| 帝国兵 解读为举盾 | 反而符合重甲兵直觉，**不用修** |

**通用 Unity 补强方案**（FE/Langrisser 标准）：
- 闪白 shader（受击瞬间全身白 0.05s）
- 后退 tween（向受击反方向位移 4-6 px，0.1s 内回弹）
- 镜头微震（amplitude 2 px, duration 0.15s）
- 红色伤害飘字（数字向上飘 + fade out 0.6s）

### 6.3 PixelLab 阶段总结
