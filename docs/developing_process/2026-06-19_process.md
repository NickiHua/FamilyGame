## 一、本 session 目标 / 背景

## 二、UI 美术：切片、去水印、归档到 Assets

在搭 uGUI 之前，先把 AI 出的 UI 图（Gemini「Nano Banana」生成的 `_alpha` 大图）切成可用 sprite 并归位。

### 2.1 切片工具 `scripts/ui/slice_sheet.py`

新写了一个**按透明间隙检测的切片器**：用 alpha 投影 profile 先切行再切列（在 `>= --min-gap` 的透明 gutter 处分割），每片紧裁、丢掉 `< --min-size` 的碎块。

```
python scripts/ui/slice_sheet.py <file|folder> [-o out] [--alpha 12] [--min-gap 10] [--min-size 12] [--pad 0]
```

对所有 `_alpha` 图跑 `--min-size 100`（顺手把 48×48 的 Gemini 水印星甩掉），输出到 `art/ui/<sheet>/<sheet>_N.png`。切出来的数量：bar_frame_states 3、bars_and_banner 3、button_states 3、command_icons 6、panel_frame 1、panels_info_and_preview 3、phase_banner 2、range_tiles 3。质量不错、边缘干净。

**坑**：`range_tiles` 的 AI 切片是**实心不透明**的（AI 画不出真 alpha），直接当格子高亮会**遮住角色** → 改用程序化的 `scripts/ui/gen_range_tiles.py`（真半透明）来做实际的 RangeOverlay。

### 2.2 去水印工具 `scripts/ui/remove_watermark.py`

Gemini 出图角落会留一个**分离的浅粉/淡紫四角星水印**。工具逻辑：在粉色像素（R−G≥20 且 B−G≥20、偏亮）上种子，然后 flood-fill 那块**连通的不透明 blob**；透明 gutter 挡住扩散，只擦掉孤立的小 blob（`--max-area 12000`）到 alpha 0。

```
python scripts/ui/remove_watermark.py <file|folder> [-o out] [--dry-run] [--max-area N] [--pink 20]
```

跑 `art/ui` → 27 张里只有 2 张带水印（bars_and_banner_2.png + phase_banner_1.png），都清掉了。

### 2.3 归档到 `Assets/Art/UI/`（语义命名）

用户确认切片后，把 `art/ui` **复制**进 `Assets/Art/UI/`（`art/ui` 原图保留作 review/版本源，**不删**），并改成语义名：

- `icons/`：icon_attack / icon_skill / icon_item / icon_wait / icon_defend / icon_move
- `bars/`：banner_frame / bar_hp / bar_mp
- `nameplates/`：nameplate_normal / nameplate_active / nameplate_dim
- `buttons/`：button_normal / button_hover / button_pressed
- `panels/`：panel_frame / panel_unit_info / panel_forecast / panel_wide
- `banners/`：banner_player（蓝）/ banner_enemy（红）
- `range/`：select_frame（覆盖成金框）+ 程序化的 move_tile / attack_tile

`PixelArtImportPostprocessor.cs` 加了 `UIRoot="Assets/Art/UI"` 规则（Single sprite、Point 滤波、PPU 100、alphaIsTransparency）。**坑**：用户必须手动跑 `Tools → FantacyCentry → Reapply Pixel Art Import Settings` 让 Unity 重新导入。

### 2.4 横幅尺寸归一化

banner_player（1315×265）和 banner_enemy（1328×308）原尺寸不一致，敌方有底部 padding 导致文字看着偏低。把两张都**裁掉透明边再 pad 到统一的 1315×276 居中画布**，覆盖回 `Assets/Art/UI/banners/`。

## 三、uGUI 五步重建（替换 OnGUI）

新建 `Assets/Scripts/View/Battle/BattleHud.cs`（`[RequireComponent(Canvas)]`），**全部 UI 用代码搭、不用 prefab**。`BattleSceneBuilder.cs` 负责建 BattleCanvas（ScreenSpaceOverlay + CanvasScaler ScaleWithScreenSize 1920×1080 match 0.5 + GraphicRaycaster）和 EventSystem（InputSystemUIInputModule，仅当没有时才建）。

- **① 金色选择框**：早就能用，无代码改动。
- **② 回合横幅**：`BattleRunner` 加 `event Action<Team,int> PhaseChanged`；横幅滑入 + 淡入淡出协程；居中、半透明（peakAlpha 0.82）；`IsBannerPlaying` **闸住回合流**（敌方 AI 必须等横幅播完才动，修了「敌人在横幅期间就动」的 bug）。
- **③ 动作菜单**：`BattleInputController` 删掉整个 `OnGUI()` HUD，暴露 `SelectedUnit` / `SelectionChanged` 事件 / `EndTurnFromUI()` / `WaitFromUI()`；HUD 建状态标签 + WAIT / END TURN 两个 9-slice 按钮（SpriteSwap）。
- **④ 伤害飘字**：删掉 `BattleRunner` 的 OnGUI 飘字，改成 HUD 上 TMP 池化飘字（世界坐标投影到 Canvas，上浮 + 淡出；MISS 灰 / 普通红 / CRIT 金）。
- **⑤ 头顶血条 + 单位信息面板**：血条选**程序化纯 Image**（不 tint 红色 bar_hp，避免脏），随单位浮在头顶、绿→黄→红、队伍色边框；信息面板用 `panel_unit_info` 像素图。

### 关键坑

- **TMP 默认字体只有拉丁字符** → UI 文字**全英文**（PLAYER PHASE / WAIT / END TURN…）。中文要等 CJK TMP 字体（还没做）。用户首次运行要 `Window → TextMeshPro → Import TMP Essential Resources`（这是**永久**资源，不是临时弹窗），不导入就「没字」。
- **TMP `enableWordWrapping` 已废弃**（CS0618）→ 改 `tmp.textWrappingMode = TextWrappingModes.NoWrap`。
- **9-slice**：按钮金角是 45° 倒角，`PixelArtImportPostprocessor` 给 buttons 设 spriteBorder (44,44,44,44)、panels/nameplates (48,48,48,48)；按钮用 `pixelsPerUnitMultiplier=2.5` 让 44px border 在 72px 高的按钮上不重叠。

## 四、战斗手感修正（玩家试玩第五步后的 3 个问题）

### 4.1 逐次掉血（敌方回合伤害不能立即结算）

**问题**：敌方回合时，我方血条**瞬间跳到最终值**，没有反映每一次攻击。

**根因**：Domain 在命令执行的瞬间就**同步**改了 `Unit.Hp`（所以敌方一波打完时，模型里所有伤害早就结算了），而 View 的视觉回放（`DrainQueue`）滞后于此。血条读的是实时 `u.Hp` → 立刻是最终值。

**修复**：在 `BattleRunner` 引入一份**滞后的「显示用 HP」**：
- `Dictionary<Unit,int> _displayHp`（spawn 时初始化为满血）+ `public int DisplayHpOf(Unit)`。
- `VisualStep` 新增 `int HpAfter`（来自 `UnitDamagedEvent.DefenderHpAfter`，入队时快照）。
- `DrainQueue` 的 Damage 分支在**这一刀视觉命中的那一刻**才 `_displayHp[step.Other] = step.HpAfter`。
- `BattleHud.UpdateHpBars` / `UpdateInfoPanel` 改读 `DisplayHpOf` 而非 `u.Hp`；血条可见性也改成由 `displayHp > 0` 驱动（不是实时 `IsAlive`），这样我方单位的血条会**保留到它的死亡动画播完**。

### 4.2 反击

**问题**：被攻击应该反击；反击过程中**不归位**（不先面朝南再转），而是直接朝对应方向反击。

**修复（Domain）**：`AttackCommand.Execute`（`BattleCommand.cs`）里，攻击结算后，若防守者存活 + 攻击者存活 + `CanCounter(Defender, Attacker)`（武器≠None 且曼哈顿距离在防守者 Min/Max 射程内）→ 防守者转向攻击者、`DamageCalc.Resolve(Defender, Attacker, …)`、扣血、**再 raise 一次 `UnitDamaged`**（攻防互换）+ 攻击者若死则 raise `UnitDied`。于是 View 自然会多回放一个 Damage 步骤。

**修复（View 顺滑）**：`DrainQueue` 用 `IsCounterOf(next, step)`（下一步是 Damage 且 Unit/Other 互换）**peek** 下一步，如果是反击就**跳过这一步之间的 FaceSouth 归位**——两人本就面对面，反击直接朝对应方向打，读起来像一次连贯的交锋，不会先「唰」地朝南再转回来。

> 注意：默认 `WeaponType = Sword(0)`，所以 EditMode 测试单位现在都会反击——但所有战斗断言仍通过。

### 4.3 信息面板放大 + 对齐（第一次尝试，后被推翻）

第一次只是把面板 `360×156 → 540×234`（1.5×），并把文字锚点**往烤死的像素图槽位上凑**。但这条路本身有问题，见第五节。

## 五、UX 转向：非像素干净风 + 信息面板彻底重建

### 5.1 用户反馈 + 讨论

用户：「战斗逻辑基本可以，这个 UX 很有问题」，希望 **UX + 立绘 + 字体都不像素风**（之前讨论过），而且 **UX 里字体要和位置对齐**，并预感「这可能不是那么简单」。

### 5.2 找到对齐问题的**根因**

不是参数没调好，而是**美术和布局耦合了**：`panel_unit_info.png` 是一张像素图，里面**烤死了**金边 / 分隔线 / 「HP」字样 / 头像框；然后运行时用 TMP 文字去**猜**这些烤死槽位的归一化坐标往上贴。所以：
- 「HP」和绿色血条**叠在一起**（图烤了 HP，我又画了个绿条）；
- 文字永远差几个像素（槽位是低分辨率像素图定的，改一次图就全错）；
- 头像框是空的（没立绘）；
- 整个像素金框与「非像素 UX」**风格冲突**。

另外澄清：**字体其实已经不是像素体**了（TMP 默认 LiberationSans，矢量干净），只是普通且没中文。真正缺的是换一个有气质的字体 + 以后加中文（Noto CJK，免费 OFL）。

### 5.3 决策（问了用户四个问题）

- **面板路线 = C**：先程序化干净面板把布局/对齐做对，HD 美术以后再换皮。
- **字体**：暂时保留 LiberationSans，先把对齐做对。
- **立绘**：先放占位（首字母），立绘以后单独生成再贴。
- **范围**：先只重做单位信息面板，验证风格后再推广。

### 5.4 信息面板重建（`BattleHud.cs`，方案 C）

**原则：美术只做装饰，uGUI 掌握布局 → 对齐由构造保证。**

- **丢掉像素面板图**（`panelUnitInfo` 字段保留但不再用，留给未来 HD 框）。
- 新增 `RoundedSprite()`：代码生成一张白色**圆角矩形 9-slice** `Texture2D`（r=16、1px 抗锯齿、border=r、静态缓存、零美术依赖）+ `NewRoundedImage(...)` helper（Sliced、白图 tint 上色），任意分辨率都干净。
- **布局**（W560 H216 pad14、右下角）：外层 = 队伍色细框；Body 暗底内缩 3px；左侧**方形头像占位**（队伍色圆角框 + 暗底 + **首字母** size96，立绘以后直接换进这个槽）；右侧容器锚到一个干净 rect，里面 —— 名字行、细分隔线、**4×2 属性网格**（ATK/DEF/MAG/RES + HIT/EVA/CRT/MOV，用 `c/4, row/2` 等分锚点 → 列必然对齐）、底部 HP 条（"HP" 标签 + 圆角槽 + 绿色填充 + 居中数字）。
- `UpdateInfoPanel`：`u = Selected ?? Hovered`，`shownHp <= 0` 时隐藏（读 `DisplayHpOf`），队伍色 tint 边框/头像。**删掉了 `_infoStats`** 那段猜锚点的多行文字。
- 面板在 `BattleHud.Awake` 现搭 → 用户**直接按 Play 就能看到**，不用重建场景。

所有改动 language-server clean。
